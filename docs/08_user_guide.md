# システムガイドライン

利用者（行政書士・登録支援機関・受入企業の担当者）向けに、本システムの目的・使い方・
留意事項・個人情報の取扱いをまとめたものです。開発者向けの詳細仕様は
[00_overview.md](./00_overview.md)以下の各ドキュメントを参照してください。

## 1. システムのタイトル

**Immigration Support Cloud（仮称）**

行政書士・登録支援機関・受入企業向けの、入管オンライン申請・特定技能定期届出支援クラウドサービス。

## 2. 目的と使用可能な機能（申請等）

### 目的

外国人材の受入れに関わる事務作業（同じ情報の何度もの転記、期限管理の属人化、
添付書類の過不足確認、定期届出のたびの前回内容の探し出しと手作業更新）を削減し、
入力ミスを減らして、誰であっても迷わずオンライン申請・定期届出の準備ができることを目指します。

### 使用可能な機能

| 機能 | 内容 |
| --- | --- |
| 顧客管理 | 法人情報・外国人情報・在留資格（種類/許可年月日/在留期限）の登録・編集・検索 |
| Excel一括取込 | 既存のExcel名簿を取り込み、内容を確認したうえでまとめて登録 |
| 添付書類管理 | 在留カード・旅券写し・雇用契約書等のPDF/画像アップロード、ファイル名自動整形、不足書類の一覧表示 |
| CSV生成 | 入管オンライン提出用CSVの自動生成（テンプレートで列構成・文字コードを管理） |
| 定期届出 | 特定技能の定期届出について、前回提出内容をコピーして差分のみ入力。面談記録・支援実施状況（10項目）の記録 |
| AI補助 | アップロード済み書類からのAIによる項目抽出（氏名・在留資格・期限等の候補提示）、ルールベースの誤入力検知（日付の前後矛盾・期限切れ・桁数不足等） |
| ユーザー管理 | 管理者によるユーザー招待・権限変更・アカウント無効化（管理者のみ） |

### 利用できるロール（権限）

| ロール | できること |
| --- | --- |
| 管理者 | 全機能。ユーザー招待・権限変更、CSVテンプレート管理 |
| スタッフ | 顧客管理・書類・CSV生成・定期届出等の実務全般（ユーザー管理は不可） |
| 閲覧のみ | 参照のみ。登録・編集・削除・ダウンロード・AI補助は利用不可 |

## 3. システムの使い方（基本的な流れ）

1. **ログイン**: メールアドレス・パスワードでログインします（アカウントは管理者からの招待メールで作成）。
2. **顧客（法人）を登録**: 「顧客管理」から法人情報を登録します。
3. **外国人情報・在留資格を登録**: 法人詳細画面から外国人を追加し、在留資格（種類・許可年月日・在留期限）を入力します。既存のExcel名簿がある場合は「Excel一括取込」でまとめて登録できます。
4. **添付書類をアップロード**: 外国人詳細画面から在留カード・旅券写し等をアップロードします。不足している必須書類は自動的に一覧表示されます。
5. **入力内容を確認**: 「入力内容のチェック」で日付の矛盾や桁数不足等が自動検出されます。「AI補助」ではアップロード済み書類からの項目抽出を利用できます（提案内容は必ず自分の目で確認してから取り込んでください）。
6. **CSVを生成 / ダウンロード**: 「入管提出用CSV生成」から、必要項目が揃っていることを確認したうえでCSVを生成・ダウンロードします。
7. **定期届出を作成**（特定技能の場合）: 「定期届出」から新しいドラフトを作成すると、前回提出内容が引き継がれます。変更点のみ入力し、面談記録・支援実施状況を記録して提出します。

## 4. 留意事項

- **AIの提案はそのまま登録されません**: AI補助機能はあくまで「提案」であり、内容を確認してチェックを入れた項目のみが反映されます。最終的な正確性の責任は利用者にあります。
- **閲覧のみロールでは操作できません**: 登録・編集・削除・ダウンロード・AI補助はスタッフ以上の権限が必要です。
- **APIキー未設定時はAI補助が使えません**: 管理者が`ANTHROPIC_API_KEY`を設定するまで、AI補助はエラーメッセージが表示され利用できません。
- **在留カード番号等の検索は完全一致のみ**: 暗号化して保存しているため、部分一致検索はできません。
- **現時点で未実装の機能**（今後対応予定）:
  - ログイン試行のレート制限・二段階認証（TOTP）
  - 個人情報操作の監査ログの実記録（画面上の権限制御自体は機能しています）
  - 期限管理ダッシュボードの通知表示（プレースホルダのみ）
- 本番運用にあたっては、ファイルストレージをローカルディスクからS3等のオブジェクトストレージへ切り替える必要があります（[07_deployment.md](./07_deployment.md)参照）。

## 5. プライバシーポリシー（概要）

本項目は社内向けの概要であり、正式な公開用プライバシーポリシーとして掲示する場合は
法務確認のうえ内容を確定してください。

- **取得する情報**: 法人情報（名称・所在地・担当者・連絡先）、外国人情報（氏名・国籍・生年月日・
  旅券番号・在留カード番号等）、在留資格・定期届出・支援実施状況に関する情報、添付書類（PDF/画像）
- **利用目的**: 入管オンライン申請・特定技能定期届出に関する書類作成・管理業務に限定して利用します。
- **保管・暗号化**: 旅券番号・在留カード番号はサーバー側でAES-256-GCM暗号化して保存し、
  通信はTLSで保護します。検索用に一方向ハッシュ（ブラインドインデックス）を別途保持しますが、
  元の値には復元できません。
- **AI機能利用時の外部送信について**: 「AI補助」機能を利用すると、アップロードした書類の画像/PDF
  データがAI処理のためAnthropic社（Claude API）に送信されます。AI補助を使わない場合、
  書類データが外部に送信されることはありません。
- **アクセス制御**: 管理者・スタッフ・閲覧のみの3段階のロールにより、参照・編集できる範囲を制限しています。
  また、事務所（テナント）単位でデータは分離されており、他の事務所のデータは参照できません。
- **第三者提供**: 上記のAI機能利用時を除き、法令に基づく場合を除いて第三者へ提供しません。
- **保存期間・削除**: 現時点で自動削除の仕組みはありません。データの削除が必要な場合は管理者にご相談ください。
- **バックアップ**: データベースは定期的にバックアップを取得し、障害時の復旧に備えています。
