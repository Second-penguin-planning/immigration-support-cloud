# プロジェクト概要

## プロジェクト名

Immigration Support Cloud（仮称）

## 背景・目的

行政書士・登録支援機関・受入企業が行う「入管オンライン申請」および「特定技能定期届出」業務は、
以下のような課題を抱えている。

- 同じ顧客情報・外国人情報を何度も転記する（Excel、紙、入管オンラインシステムへの再入力）
- 在留期限・届出期限の管理が担当者の記憶やExcelの手動チェックに依存している
- 添付書類の過不足確認が目視頼りでミスが起きやすい
- 定期届出のたびに前回提出内容を探し出し、差分だけを手作業で更新している

本サービスは、これらの入力作業を **80%以上削減**し、人的ミスを減らし、
行政書士・登録支援機関・受入企業の誰であっても迷わずオンライン申請・定期届出の準備ができることを
目指す。

## 対象ユーザー

| ユーザー種別 | 主な利用シーン |
| --- | --- |
| 行政書士 | 複数の受入企業・外国人の申請書類を一括管理し、入管提出用データを作成する |
| 登録支援機関 | 特定技能外国人の支援実施状況を記録し、定期届出を作成する |
| 受入企業 | 自社の外国人従業員の在留資格・期限を把握し、行政書士/登録支援機関と情報共有する |

## MVPスコープ（本ドキュメント群が対象とする範囲）

1. ログイン機能（メール認証・パスワードリセット・権限管理）
2. 顧客管理（法人情報・外国人情報・在留資格・期限・担当者・検索・CSVダウンロード）
3. Excelアップロード（自動解析・必要項目抽出・DB保存・入力エラー表示）
4. CSV生成（入管オンライン提出用CSVの自動生成、テンプレート差し替え可能な構造）
5. PDF管理（添付書類アップロード・ファイル名自動変更・不足書類表示・書類一覧表示）
6. 定期届出（前回データコピー・変更点のみ入力・面談記録・支援実施状況・届出ドラフト作成）
7. AI補助（OCR・PDF読取・不足項目抽出・誤入力検知・入力候補表示）
8. ダッシュボード（期限管理: 30日/14日/7日/当日、通知表示）

## MVP後に追加予定（本フェーズでは設計・実装しない）

顧客ポータル、電子署名、請求書発行、Stripe決済、LINE通知、メール自動送信、チャット、外部API連携。
これらはMVPのデータモデル・APIを不必要に複雑化させないよう、Phase8完了後に個別検討する。
